## Executive Summary

The NGO Treasury application demonstrates **solid foundational security** — JWT validation, RBAC, parameterized Cosmos DB queries, and Pydantic input validation are all well-implemented. However, the scan identified **significant gaps** in infrastructure hardening, rate limiting, security headers, and CI/CD hygiene that must be addressed before production.

Switch's expert review adjusted several severity levels and identified 5 additional findings missed by the generalist scan.

### Finding Counts (Post-Review)

### Positive Findings (No Action Needed)
- Parameterized Cosmos DB queries — no injection vectors
- Pydantic validation on all API inputs — no mass assignment
- JWT signature verification with JWKS
- Non-root Docker container user
- MSAL route guards on all protected routes
- No XSS patterns (innerHTML, bypassSecurityTrust) in Angular
- PII logging disabled in MSAL
- Managed Identity + RBAC for Cosmos DB and Key Vault
- HTTPS-only, TLS 1.2+, FTPS disabled on App Service

## CRITICAL Findings (5)

### C1. Overly Permissive CORS
- **OWASP:** A04 — Insecure Design
- **Files:** `api/app/main.py` (L37-42), `infra/modules/app-service.bicep` (L60-66)
- **Issue:** `allow_methods=["*"]`, `allow_headers=["*"]` combined with `allow_credentials=True`.
- **Fix:** Restrict to `["GET", "POST", "PUT", "DELETE"]` and `["Content-Type", "Authorization"]`.

### C2. No Content Security Policy (Frontend)
- **OWASP:** A05 — Security Misconfiguration
- **Files:** `frontend/src/index.html`, `frontend/src/staticwebapp.config.json`
- **Issue:** No CSP meta tag or response headers. Inline scripts can execute; no frame-ancestors restriction.
- **Fix:** Add CSP to `staticwebapp.config.json` via `globalHeaders` and add security headers (X-Frame-Options, X-Content-Type-Options, HSTS, Referrer-Policy).

### C3. Cosmos DB — Local Auth Enabled + No Network Restrictions
- **OWASP:** A05 — Security Misconfiguration
- **File:** `infra/modules/cosmos-db.bicep` (L38)
- **Issue:** `disableLocalAuth: false` — primary keys can bypass RBAC. No IP firewall or private endpoint configured; Cosmos DB is publicly routable.
- **Fix:** Set `disableLocalAuth: true`. Add IP firewall rules or private endpoint.

### C4. COSMOS_KEY in GitHub Actions Deployment
- **OWASP:** A07 — Identification and Authentication Failures
- **File:** `.github/workflows/deploy-prod.yml` (L115)
- **Issue:** Cosmos DB key passed as GitHub secret and set as app setting. Permanent credential — if leaked, full DB access.
- **Fix:** Remove COSMOS_KEY; use Managed Identity with `DefaultAzureCredential` exclusively.

### C5. File Upload Content-Type Validation Incomplete *(upgraded from HIGH by Switch)*
- **OWASP:** A04 — Insecure Design
- **File:** `api/app/routers/imports.py` (L21-27)
- **Issue:** Accepts `application/octet-stream` without magic byte validation. Combined with lack of `defusedxml`, openpyxl ZIP bomb is a real resource exhaustion vector.
- **Fix:** Restrict to Excel MIME types; validate ZIP magic bytes (`PK\x03\x04`); install `defusedxml`.
- **Switch note:** Upgraded because `octet-stream` bypass + ZIP bomb = actionable exploit path. See also M10.

## HIGH Findings (11)

### H1. Missing Rate Limiting *(downgraded from CRITICAL by Switch)*
- **OWASP:** A04 — Insecure Design
- **File:** `api/app/main.py` (missing middleware)
- **Issue:** No rate limiting on any endpoint. Enables DoS and resource exhaustion (especially imports, exports, reports).
- **Fix:** Add `slowapi` with per-endpoint limits (e.g., 100/min general, 5/min imports).
- **Switch note:** Downgraded because all endpoints require Entra ID JWT — brute-force is low threat for this NGO app. Still important, but not production-blocking.

### H2. App Insights — Public Network Access *(downgraded from CRITICAL by Switch)*
- **OWASP:** A05 — Security Misconfiguration
- **File:** `infra/modules/app-insights.bicep` (L39-40)
- **Issue:** `publicNetworkAccessForIngestion: 'Enabled'`, `publicNetworkAccessForQuery: 'Enabled'`. Anyone with the instrumentation key can submit telemetry or read diagnostics.
- **Fix:** Set both to `'Disabled'`; use private endpoint or VNet integration.
- **Switch note:** Telemetry poisoning is real but lower-impact than database access. HIGH, not CRITICAL.

### H3. JWT Error Details Leaked in Response
- **OWASP:** A01 — Broken Access Control
- **File:** `api/app/auth/dependencies.py` (L107)
- **Issue:** `detail=f"Token validation failed: {str(last_error)}"` — reveals which validation checks fail.
- **Fix:** Return `"Invalid or expired token"`. Log details server-side.

### H4. Missing Security Response Headers (Backend API)
- **OWASP:** A05 — Security Misconfiguration
- **File:** `api/app/main.py`
- **Issue:** No HSTS, X-Content-Type-Options, X-Frame-Options, Referrer-Policy headers.
- **Fix:** Add `SecurityHeadersMiddleware` to FastAPI.

### H5. JWKS Cache Race Condition
- **OWASP:** A06 — Vulnerable and Outdated Components
- **File:** `api/app/auth/dependencies.py` (L8-31)
- **Issue:** Global variables without async lock; old keys valid for 1 hour after rotation.
- **Fix:** Add `asyncio.Lock()`; invalidate cache on unknown `kid`.

### H6. Key Vault — No Purge Protection
- **File:** `infra/modules/key-vault.bicep` (L27-35)
- **Issue:** Soft delete enabled but `enablePurgeProtection` not set. Secrets can be permanently deleted.
- **Fix:** Add `enablePurgeProtection: true`.

### H7. Static Web App — Config File Updates Enabled
- **File:** `infra/modules/static-web-app.bicep` (L24)
- **Issue:** `allowConfigFileUpdates: true` — runtime API can modify security headers/routing.
- **Fix:** Set to `false`.

### H8. GitHub Actions — Permissions Not Explicitly Defined
- **Files:** `.github/workflows/deploy-prod.yml`, `.github/workflows/deploy-infra.yml`
- **Issue:** No `permissions` block → defaults to `write-all`.
- **Fix:** Add explicit minimal permissions (`contents: read`, `id-token: write`).

### H9. GitHub Actions — Third-Party Actions Not Pinned to SHA
- **Files:** All `.github/workflows/*.yml`
- **Issue:** Actions pinned to floating tags (`@v4`), not commit SHA. Supply chain risk.
- **Fix:** Pin all actions to commit SHA.

### H10. Azure Credentials — Not Using OIDC
- **File:** `.github/workflows/deploy-prod.yml`
- **Issue:** `AZURE_CREDENTIALS` is a permanent JSON service principal secret. OIDC tokens auto-rotate.
- **Fix:** Migrate to federated credential + OIDC login.

### H11. `python-jose==3.3.0` Unmaintained *(NEW — found by Switch)*
- **OWASP:** A06 — Vulnerable and Outdated Components
- **File:** `api/requirements.txt`
- **Issue:** `python-jose` last released in 2021, has known issues, and is effectively abandoned. PyJWT is already installed as a transitive dependency via MSAL.
- **Fix:** Replace `python-jose` with `PyJWT[crypto]`; update `api/app/auth/dependencies.py` to use `jwt` instead of `jose.jwt`.

## MEDIUM Findings (10)

### M1. Tokens Stored in localStorage (Frontend)
- **File:** `frontend/src/app/core/auth/msal.config.ts` (L20-24)
- **Issue:** `BrowserCacheLocation.LocalStorage` — XSS can steal tokens. Tokens persist across sessions.
- **Fix:** Switch to `BrowserCacheLocation.SessionStorage`.

### M2. File Import — No Magic Byte / Size Validation on Frontend
- **File:** `frontend/src/app/features/import/import.component.ts`
- **Issue:** Only `accept=".xlsx"` attribute (client hint, bypassable). No MIME type or size check.
- **Fix:** Validate MIME type and enforce 10 MB limit client-side.

### M3. Export Date Range Unbounded
- **File:** `api/app/routers/export.py` (L9-27)
- **Issue:** No validation that `dateFrom <= dateTo` or max range. Could trigger massive Cosmos DB scans.
- **Fix:** Validate date ordering; cap at 90 days.

### M4. Financial Calculations Use Float
- **File:** `api/app/services/transaction_service.py` (L62-67)
- **Issue:** `abs(float(data.amount))` — loses Decimal precision. Rounding errors accumulate.
- **Fix:** Keep as `Decimal` during arithmetic; convert to float only for Cosmos DB storage.

### M5. App Insights Retention Only 30 Days
- **File:** `infra/modules/app-insights.bicep` (L23)
- **Issue:** NGO compliance may require longer audit trail retention.
- **Fix:** Extend to 365 days.

### M6. App Service — Missing Diagnostic Logging
- **File:** `infra/modules/app-service.bicep`
- **Issue:** No diagnostic settings to send logs to Log Analytics.
- **Fix:** Add diagnostic settings resource.

### M7. Insufficient Logging of Auth Failures *(upgraded from LOW by Switch)*
- **Files:** `api/app/auth/dependencies.py`, `api/app/middleware/error_handler.py`
- **Issue:** Failed authentication and authorization not explicitly logged.
- **Fix:** Add `logger.warning()` for 401/403 events.
- **Switch note:** Upgraded because without auth failure logging, credential stuffing detection is impossible.

### M8. Swagger/OpenAPI Exposed in Production *(NEW — found by Switch)*
- **OWASP:** A05 — Security Misconfiguration
- **File:** `api/app/main.py`
- **Issue:** `/api/docs` (Swagger UI) and `/api/openapi.json` serve the full API schema without authentication in production. Gives attackers a complete endpoint map.
- **Fix:** Disable in production: `docs_url=None, openapi_url=None` when `ENVIRONMENT == "production"`.

### M9. PR Tests Suppressed with `|| true` *(NEW — found by Switch)*
- **OWASP:** A08 — Software and Data Integrity Failures
- **File:** `.github/workflows/pr-checks.yml`
- **Issue:** Test step uses `|| true`, meaning test failures don't block PR merges. Broken code can ship.
- **Fix:** Remove `|| true`; let test failures fail the workflow.

### M10. openpyxl ZIP Bomb Surface *(NEW — found by Switch)*
- **OWASP:** A04 — Insecure Design
- **File:** `api/requirements.txt`
- **Issue:** `defusedxml` is not installed. Without it, openpyxl's XML processing has no decompression bomb protection.
- **Fix:** `pip install defusedxml`; add to `requirements.txt`. openpyxl uses it automatically when present.

## LOW Findings (7)

### L1. Audit Log TTL — 7 Years Fixed
- **File:** `api/app/services/audit_service.py` (L14)
- **Issue:** `_AUDIT_TTL = 220752000` (7 years). Should be configurable; some NGO compliance requires indefinite retention.
- **Fix:** Make configurable via environment variable.

### L2. Token Validation Retry — Overly Generous
- **File:** `api/app/auth/dependencies.py` (L72-84)
- **Issue:** 4 audience/issuer combinations tried. Complex to audit; silently accepts v1 tokens.
- **Fix:** Document expected token format; consider strict v2-only validation.

### L3. Frontend Environment Config in Source Control
- **File:** `frontend/src/environments/environment.prod.ts`
- **Issue:** Client ID, tenant ID committed. Technically public for SPAs, but enables reconnaissance.
- **Note:** Acceptable for MSAL SPAs. Low priority.

### L4. Dependency Caret Versioning (Frontend)
- **File:** `frontend/package.json`
- **Issue:** `^` allows minor/patch updates automatically.
- **Fix:** Use `npm ci` in CI; consider exact pinning.

### L5. No CSRF Protection *(downgraded from MEDIUM by Switch)*
- **Files:** All POST/PUT/DELETE endpoints in `api/app/routers/`
- **Issue:** No CSRF tokens. Mitigated by Bearer token auth.
- **Switch note:** Non-issue for Bearer-token APIs — CSRF attacks require cookie-based auth. Document the threat model decision and move on.

### L6. Import Creates Categories Without Approval *(downgraded from MEDIUM by Switch)*
- **File:** `api/app/services/import_service.py` (L475-492)
- **Issue:** Admin can import Excel with arbitrary category names; categories auto-created.
- **Switch note:** Admin-only feature with trusted users. This is a business process question, not a security vulnerability.

### L7. No Logging of Cosmos Auth Method *(NEW — found by Switch)*
- **File:** `api/app/services/cosmos_client.py`
- **Issue:** No logging of whether Cosmos DB connection uses key-based auth or Managed Identity. Makes it hard to verify that production is actually using Managed Identity.
- **Fix:** Log the auth method at startup: `logger.info("Cosmos DB auth: Managed Identity" if not key else "key-based")`.
