- (2026-04-18) **System Settings (Issue #12) — backend scoping.** Singleton-config pattern in Cosmos: reuse the existing `reference_data` container with `id="system"`, `partitionKey="system_settings"`. Avoids RU floor cost of a new container; matches `accounts`/`tags` precedent of using `type` as a discriminator. Lookup is a point-read (1 RU). Default seeding via lazy-create on first GET — no startup writes, runs at most once per deployment.
- (2026-04-18) **A1 (server-authoritative audit) — two-layer defense:** Pydantic `model_config = ConfigDict(extra="forbid")` on the update DTO rejects any client-supplied `updatedAt`/`updatedBy` with HTTP 422 before the handler runs. Service-side, always overwrite from server-supplied parameters (`current_user["oid"]` from `Depends(get_current_admin)`, `datetime.now(timezone.utc)`). Even if schema validation were ever loosened, the service would still overwrite. Use Entra `oid` as the immutable primary identifier.
